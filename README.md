A modern, production-ready **Single Page Application (SPA)** simulating a YouTube subscriber exchange platform. Built with HTML5, Tailwind CSS v4, and Vanilla JavaScript — featuring a Sub-to-Unlock security gateway, virtual credit economy, and glassmorphism UI.

> **Live Demo:** [https://your-username.github.io/subxchange-pro/](https://your-username.github.io/subxchange-pro/)

---

## ✨ Features

- 🔒 **Sub-to-Unlock Gateway** — Mandatory admin subscription with 15s verification timer
- 💰 **Virtual Credit Economy** — Earn +5 credits per subscription, spend 10 credits to promote your channel
- 📊 **Live Dashboard** — Real-time stats, community pool, leaderboards, and campaign analytics
- 🎁 **Daily Rewards & VIP Tier** — Check-in bonuses and 2x multiplier upgrades
- 🛡️ **Anti-Cheat Simulation** — Focus tracking and verification countdowns
- 📱 **Fully Responsive** — Mobile-first design that works on any device
- 🌙 **Dark-Mode Glassmorphism** — Modern SaaS-style aesthetic with smooth animations

## 🚀 Quick Start (Local)

1. **Clone the repository:**
   ```bash
   git clone https://github.com/your-username/subxchange-pro.git
   cd subxchange-pro
   ```

2. **Open in browser:**
   Simply double-click `index.html` or use a live server:
   ```bash
   # Using Python
   python -m http.server 8000
   
   # Using Node.js
   npx serve .
   ```

3. **Visit:** `http://localhost:8000`

## 🌐 Deploy to GitHub Pages

### Option A: Manual (Easiest)

1. Create a new **public** GitHub repository named `subxchange-pro`
2. Upload `index.html` and `README.md` to the repo
3. Go to **Settings** → **Pages**
4. Under **Source**, select `main` branch and `/ (root)` folder
5. Click **Save** — your site will be live at `https://your-username.github.io/subxchange-pro/`

### Option B: Git Commands

```bash
# Initialize repo
git init
git add .
git commit -m "🎉 Initial commit: SubXchange Pro"

# Link to GitHub (replace with your repo URL)
git remote add origin https://github.com/your-username/subxchange-pro.git
git branch -M main
git push -u origin main
```

Then enable GitHub Pages in repository settings.

## 🎯 How It Works

1. **Gateway Lock** → User must subscribe to admin channel (Rushi Lofi)
2. **15s Verification Timer** → Ensures genuine engagement
3. **Dashboard Access** → Unlocked state saved to localStorage
4. **Earn Credits** → Subscribe to other creators' channels (+5 credits each)
5. **Promote Channel** → Spend credits to add your channel to the community pool
6. **Receive Subscribers** → Other users subscribe to you, fulfilling your campaign

---

## ⚠️ Important Notes

- **Demo Purpose:** This is a **frontend simulation**. Data is stored per-browser in localStorage, not shared across users.
- **Production Upgrade:** To make it a real multi-user platform, integrate Firebase/Supabase for authentication and a shared database.
- **YouTube ToS:** Subscriber exchange platforms may violate YouTube's Terms of Service regarding artificial engagement. Use responsibly as a learning/portfolio project.
